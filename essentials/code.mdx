Ziplime использует событийную модель Zipline: движок последовательно обрабатывает рыночные бары, вызывает функции жизненного цикла стратегии и моделирует заявки, позиции и стоимость портфеля.

## Что даёт движок

1. **Событийное исполнение** — код получает данные в хронологическом порядке.
2. **Минутные и дневные данные** — доступный набор частот зависит от источника и инструмента.
3. **Асинхронный API** — получение данных и торговые операции выполняются через async-функции.
4. **Расширяемость** — можно добавлять собственные индикаторы, факторы и метрики.
5. **Единая логика** — один алгоритмический файл можно использовать как основу для бэктеста и подготовленного live-сценария.

## Минимальный пример

~~~python
from ziplime.finance.execution import MarketOrder


async def initialize(context):
    context.asset = await context.symbol("SBER")
    context.invested = False


async def handle_data(context, data):
    if not context.invested:
        await context.order_target_percent(
            asset=context.asset,
            target=1.0,
            style=MarketOrder(),
        )
        context.invested = True
~~~

Имена функций и сигнатуры API не переводятся. Полный справочник находится в разделе [«Язык стратегий»](/language/strategy-language/overview).

<Warning>
  Пример показывает структуру алгоритма, а не готовую инвестиционную стратегию.
</Warning>
